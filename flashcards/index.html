3. Flow, Conditionals & Arrays

Do you know the best way to iterate through the items of an array?

With #each:

[1, 2, 3].each do |num|
  puts num

What syntax would you rather use when you have many cases in your conditional statement?

You'd rather use case when statements:

num = 42
case num
when 1
  puts "one"
when 2
  puts "two"
when 42
  puts "the answer!"
else
  puts "nope"
end

Don't forget the else statement at the end if you want to be exhaustive!

How can you delete an item from an array?

You can call #delete on the array, passing it the item you want to delete as an argument

beatles = ["John", "Ringo", "Paul", "George"]

You can also call #delete_at on the array, passing it the index of item you want to delete as an argument

Which iterator should you call on an Array to get another Array where all the elements were subject to the same treatment?

How would you define a method accepting a block with parameters?

You need to use yield(parameters).

def greet(name)
  capitalized_name = name.capitalize
  puts yield(capitalized_name)
end

me = "john"

greet(me) do |name|
  "Greetings, #{name}, you look quite fine today!"
end

# => "Greetings, John, you look quite fine today!"

What is a destructive method? Give an example.

A variable on which is called a destructive method is modified forever.

By convention, destructive methods end with a ! to warn it's dangerous, like #capitalize! or #map!

If you want to capitalize a string:

name = "john"

name.capitalize!

puts name

# => "John"

If you want to capitalize an array of strings:

names = ["JOHN", "paul", "Ringo", "gEOrgEs"]

names.map! do |name|
  name.capitalize
end

puts names

# => ["John", "Paul", "Ringo", "Georges"]

In Ruby there are two kinds of iterators, those who return the transformed array, and those who don't. Can you name iterators from both categories?

5. Hash & Symbols

What is the difference between reading values from a Hash and from an Array?

  "name" => "Paris",
  "population" => 2211000
}

cities[0]# => "London"
city["name"] # => "Paris"

What is the difference between a Symbol and a String?

A Symbol is what we call an identifier. An instance of Symbol occupies one spot in the memory of your computer.
Every time it's called in your code, it refers to the unique piece of text defined earlier. 

hello_sym_one = :hello
hello_sym_two = :hello

hello_sym_one.object_id
#=> 2376028

hello_sym_two.object_id
#=> 2376028

As a contrary, two String instances with the same text fill two different spots in the memory of your computer.

hello_string_one = "hello"
hello_string_two = "hello"

hello_string_one.object_id
#=> 70246165778340

hello_string_two.object_id
#=> 70246154320020

Why do we need IDs in our models?

We need them to translate relations between tables in a database.

In a table's row, a reference to another table's record is called a foreign key.

We don't need them in the objects world, as we can store objects in instance variables, but we have no choice but to use id's to track references between records from different tables in a database!

Explain why databases auto increment IDs

When a new entry is created, a new unique number is assigned to its id.
Once an id has been consumed, it should never be given to another record, even if the record was deleted since (that way we are sure the id really identifies the record it was assigned to).
This is a common mechanism in databases, and this is why most engines do it automatically!
